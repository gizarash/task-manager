# task-manager

## Этап 1: CLI Task Manager

### Задание

Напиши консольную программу управления задачами. Программа должна поддерживать следующие команды:

```
go run main.go add "Купить молоко"
go run main.go list
go run main.go done 1
go run main.go delete 1
```

### Требования

add <текст> — добавляет новую задачу  
list — выводит все задачи с ID, статусом и текстом, например:
```
[1] [ ] Купить молоко
[2] [x] Позвонить маме
```
done \<id> — отмечает задачу как выполненную  
delete \<id> — удаляет задачу

Если команда неизвестна или аргументы неверны — вывести понятное сообщение об ошибке.

### Ограничения этапа

* Хранение только в памяти (данные не сохраняются между запусками — это будет этап 2)
* Всё в одном файле main.go
* Не использовать сторонние библиотеки

### На что обратить внимание

* Как ты определишь структуру задачи?
* Как будешь хранить список задач?
* Как будешь разбирать аргументы командной строки?

## Этап 2: Сохранение в JSON-файл

Теперь данные должны сохраняться между запусками. Команды остаются теми же, но теперь:

```
go run . add "Купить молоко"   # сохранить в файл
go run . list                  # прочитать из файла и вывести
go run . done 1                # обновить файл
go run . delete 1              # обновить файл
```

### Задание

Добавь персистентность через JSON-файл tasks.json в текущей директории.

### Логика работы:

1. При старте — читать задачи из файла (если файл не существует — начинать с пустого списка)
2. После каждой изменяющей операции (add, done, delete) — записывать актуальное состояние обратно в файл

### Требования

* Файл tasks.json создаётся автоматически при первом add
* currentId тоже должен сохраняться (чтобы ID не начинался с 1 после каждого запуска)
* Использовать только стандартную библиотеку: encoding/json, os

### На что обратить внимание

* Как читать файл, которого ещё не существует?
* Как отличить "файл не существует" от других ошибок чтения?
* Где лучше хранить currentId — в том же файле или считать на основе имеющихся задач?
